# SCENE_INTERACTION_SPEC: 配给日

## Scene Layout

```
+----------------------------------------------+
| 顶部状态栏                                    |
| Day X  |  剩余口粮: XXXg  |  怀疑: XX/100     |
+----------------------------------------------+
|                                              |
|  [口粮堆]        NPC 称盘区                   |
|  可拖动粮块      [老周秤] [小陈秤] [张姐秤] [赵六秤]|
|                  显示已放克数和 hunger/trust    |
|                                              |
+----------------------------------------------+
|  队列/事件区域                                 |
|  NPC 对话气泡、请求、反应文本                    |
|  [关系预览]  [确认发放]                         |
+----------------------------------------------+
```

## Scene Objects

### 1. 口粮堆（Food Pile）
- 位置：界面左侧
- 外观：堆叠的粮块，数量 = 剩余 food / 50（每块代表 50g）
- 行为：玩家从口粮堆拖动粮块到 NPC 称盘
- 视觉反馈：拖动时粮块跟随鼠标；放入称盘后口粮堆减少对应块数

### 2. NPC 称盘（Character Scales）×4
- 位置：界面中排，4 个并排
- 每个称盘显示：
  - NPC 名字和头像占位
  - 当前已放克数（大字显示）
  - hunger 条（红→绿，数值标注）
  - trust 条（灰→蓝，数值标注）
- 行为：
  - 拖入粮块：克数增加 50g
  - 点击已放粮块：弹出 ±50g 调整（可连续点击）
  - 拖出粮块到口粮堆：克数减少 50g
- 视觉反馈：
  - 克数 > 公平线：称盘微微下沉，绿色高亮
  - 克数 < 公平线：称盘微微上翘，黄色警告
  - 克数 = 0：灰色空盘

### 3. 队列/事件区（Queue Zone）
- 位置：界面底部
- 显示内容：
  - NPC 排队顺序（影响事件触发优先级）
  - Phase B：NPC 请求气泡（"能不能多给点？"）
  - Phase C：工头警告文本
  - Phase D：危机通知
- 行为：纯文本展示，不需交互

### 4. 关系预览面板（Preview Panel）
- 位置：确认发放按钮上方
- 内容：
  - 每人称量差额（vs 公平线）
  - 预计 hunger 变化方向：↑ +N / ↓ -N / → 不变
  - 预计 trust 变化方向：↑ +N / ↓ -N / → 不变
  - 预计 suspicion 变化：↑ +N / → 不变（Day 5-6 才显示）
- 实时更新：每次称盘克数变化时刷新预览

### 5. 确认发放按钮（Confirm Button）
- 位置：预览面板下方
- 启用条件：至少 2 个称盘有粮块
- 点击后：触发结算（调用 MECHANIC_SPEC 公式）
- 结算后：显示结果反馈 2 秒，然后进入下一日或结局

## Feedback Channels

### 即时反馈（操作时）
- 称盘重量数字变化
- 口粮堆块数减少
- 公平线对比色变

### 结算反馈（确认后）
- 每人 hunger 数值变化动画（数字滚动）
- 每人 trust 数值变化动画
- suspicion 变化（如有）
- NPC 反应文本（1 句话，从内容池选取）
- 崩溃条件触发时：红色警告 + 失败原因

### Day Transition
- 简短过渡：Day X → Day X+1
- 新日事件文本（如有）

## Drag-and-Drop 规格

- 拖动源：口粮堆中的粮块（每块 50g）
- 拖动目标：NPC 称盘
- 拖动取消：释放在非称盘区域，粮块回到口粮堆
- 精细调整：点击称盘中粮块弹出 ±50g 步进器
- 最大克数：当日 food 总量
- 最小克数：0g（可以不给某人）

## Forbidden UI

- 不允许只用"给 A / 给 B / 不给"文字按钮
- 不允许做餐厅经营菜单
- 不允许用下拉菜单选择克数
- 不允许跳过预览直接结算

## Acceptance Rules

- 首屏必须让玩家看到口粮堆和至少 4 个称盘
- 拖动粮块到称盘必须在 300ms 内产生可见反馈
- 预览面板必须在克数变化后 500ms 内更新
- 所有反馈必须可追溯到 Required State 中的具体字段
- 不得只靠随机事件文本或普通选择按钮完成主循环
