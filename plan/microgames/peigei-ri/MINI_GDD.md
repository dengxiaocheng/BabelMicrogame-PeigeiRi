# MINI_GDD: 配给日

## Scope

- runtime: web (单页 HTML + JS/CSS)
- duration: 20min
- project_line: 配给日
- single_core_loop: 查看人物饥饿和队列 -> 称量口粮 -> 分发 -> 结算饱腹/信任/怀疑 -> 下一日

## NPC Roster（4 人固定）

| ID | 名字 | 人设 | 饥饿倾向 | 关系倾向 |
|----|------|------|----------|----------|
| laozhou | 老周 | 老实工人，沉默寡言 | 不主动讨要 | 信任稳定，被骗后暴降 |
| xiaochen | 小陈 | 年轻工人，活泼爱聊 | 会暗示不够吃 | 给多了快速拉 trust |
| zhangjie | 张姐 | 带孩子的母亲 | 明确说不够 | 给少了 trust 暴降 |
| zhaoliu | 赵六 | 精明，常走后门 | 会私下请求多给 | 偏私后替你说话（降 suspicion） |

## Day Pacing（8 天 / 4 阶段）

### Phase A — 正常分粮（Day 1-2）
- 每日口粮池：1200g（公平线 300g/人）
- 目的：教会称量拖动操作
- 无额外事件压力

### Phase B — 熟人插队（Day 3-4）
- 口粮池：1200g
- 1-2 名 NPC 主动请求额外口粮（对话气泡）
- 拒绝请求：该 NPC trust -5；答应：其他 NPC 微涨 suspicion +3

### Phase C — 工头抽查（Day 5-6）
- 口粮池：1200g
- 工头出现，检查分配是否公平
- 分配方差过大（最大 - 最小 > 150g）：suspicion +10
- 方差适中：suspicion 不变

### Phase D — 口粮减半危机（Day 7-8）
- 口粮池：600g（公平线 150g/人）
- 不可能让所有人满意
- 考验前期积累的 trust 缓冲
- 结算日：Day 8 结束后进入结局

## State 初始值

```
day = 1
food = 1200 (Day 7-8 改为 600)
suspicion = 0
hunger = { laozhou: 20, xiaochen: 25, zhangjie: 30, zhaoliu: 20 }
trust   = { laozhou: 50, xiaochen: 50, zhangjie: 50, zhaoliu: 50 }
```

## UI Zones

```
+------------------------------------------+
| Day X | 剩余口粮: 1200g | 怀疑: 0/100    |  <- 状态栏
+------------------------------------------+
|  口粮堆   |  称盘区域                    |
|  [粮块]   |  [老周?] [小陈?] [张姐?] [赵六?] |  <- 拖动目标
|  [粮块]   |   300g    200g    350g   150g  |
+------------------------------------------+
| 队列/对话区域                             |  <- NPC 反馈
| [确认发放]                                |  <- 操作按钮
+------------------------------------------+
```

## Content Scope

- NPC 对话池：每阶段 2-4 条，按分配结果触发
- 事件类型：请求加量、暗示不满、工头警告、危机通知
- 不做复杂分支叙事；每次事件只影响数值
- 总文本量控制在 100 行以内

## Constraints

- 总体规模目标控制在 5000 行以内
- 单个 worker 任务必须服从 packet budget
- 如需扩线，交回 manager 重新拆
